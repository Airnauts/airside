# Airside Rebrand Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebrand the whole product from the `comments` brand/namespace to **Airside** — `@airnauts/comments-*` → `@airnauts/airside-*`, plus every brand-carrying token (CSS prefix, data-attrs, storage keys, query params, header, env vars, public JS symbols, repo URLs, docs) — landing as one breaking minor (`0.8.0`), with the old npm packages deprecated to point at the new ones.

**Architecture:** A wide, **token-scoped** rename executed as an ordered sequence of green commits on branch `worktree-airside-rebrand-spec`. Each commit renames one token family (source + its test literals together) and leaves `pnpm -w build && pnpm -w test` green. Ordering is load-bearing: families whose tokens are *substrings* of other families are renamed first (storage keys before the Tailwind `cmnt:` sweep; `data-comments-` before the `comments-…` test-ids; the 3 extension packages before the generic `@airnauts/comments-` sweep). The one publish path is Changesets + CI; nothing is published by hand.

**Tech Stack:** pnpm workspaces · Changesets (fixed group, pre-1.0 policy: breaking → **minor**) · Tailwind v4 (prefix-namespaced widget CSS, regenerated at build) · tsup + `tsc --build` · Vitest · Playwright (e2e) · GitHub Actions auto-publish.

Source of truth: `docs/superpowers/specs/2026-06-15-airside-rebrand-design.md` (ADR to be recorded as **ADR-0038**).

## Conventions used by every task

### The standard sweep idiom

All mechanical replacements use `git grep -l` to build the file list. This is deliberate — `git grep` only sees **tracked** files, so it automatically skips `node_modules/`, `dist/`, and the **gitignored** generated CSS (`packages/client/src/app/widget-css.generated.ts`, regenerated by `pnpm build:css`). On top of that, every sweep carries the **standard exclusion set** so we never rewrite history:

```
:!**/CHANGELOG.md      # release history — those entries correctly say comments-* (they shipped under that name)
:!docs/adr.md          # decided ADR history — ADR-0038 supersedes ADR-0020, so ADR-0020 must stay verbatim
:!docs/superpowers/**  # specs/plans describe the from→to rename; rewriting them corrupts their meaning
```

Canonical form (substitute `OLD`/`NEW`; `perl` brace-delimiters `s{…}{…}g` avoid escaping `/`):

```bash
git grep -l 'OLD' -- ':!**/CHANGELOG.md' ':!docs/adr.md' ':!docs/superpowers/**' \
  | xargs perl -pi -e 's{OLD}{NEW}g'
```

### Standard verification (after every code task)

```bash
pnpm -w build && pnpm -w test
```

`pnpm -w build` is required even for "client-only" changes because consumers resolve dependencies through built `dist/` (the worktree uses **real** node_modules, not symlinks — see the worktree-setup note in memory), and `client` imports `@airnauts/comments-core`. Tasks that touch only `packages/client` may scope the *test* run to `pnpm --filter "./packages/client" test` for speed, but still run `pnpm -w build` first.

### DO NOT TOUCH — domain survivors

These name *the thing the product manages*, not the brand. They MUST remain after every sweep:

- `Comment` type / zod schema (singular, capitalized).
- `comments[]` data fields, `commentCount`.
- the `comments` **Mongo database name** inside `MONGODB_URI`.
- UI copy: "Add comment", "Reply".
- `ServerExtension` / `NotificationExtension` types (domain seam, not brand).

Because of these, two tokens are **never** mass-swept: the bare lowercase word `comments` and the bare capitalized `Comments`. They are hand-edited at enumerated sites only (Task 9).

```markdown
## ADR-0038 — Rebrand `comments` → `airside`

**Date:** 2026-06-16
**Status:** accepted (supersedes the package-naming portion of ADR-0020)

**Context.** The product was published under the generic `comments` brand (`@airnauts/comments-*`, repo `commenting-tool`, `cmnt:` CSS prefix). "comments" is both our brand *and* the domain noun, which muddied identity and SEO and tied us to a generic name. While still pre-1.0 (one production consumer), a clean break is cheap. "Airside" plays on "aside" (a remark to the side) and the aviation term, tying to the Airnauts brand.

**Decision.** Full rebrand of the brand/namespace, keeping the npm scope `@airnauts`:
- Packages `@airnauts/comments-*` → `@airnauts/airside-*`; the three extension packages are recategorized to `@airnauts/airside-extension-{slack,email,jira}`.
- Every brand-carrying token moves: Tailwind prefix `cmnt:` → `air:`, CSS vars `--cmnt-*` → `--air-*`, `data-comments-*` → `data-airside-*`, storage keys `comments:*`/`cmnt:focus` → `airside:*`, query params `comments-key`/`comments-thread` → `airside-*`, header `x-comments-key` → `x-airside-key`, env `COMMENTS_*` → `AIRSIDE_*`, public JS symbols (`comments.init`→`airside.init`, `<CommentsLayer>`→`<AirsideLayer>`, `CommentsHandle`→`AirsideHandle`, `commentsKey`→`airsideKey`, `createComments*Route`→`createAirside*Route`, `CommentsServer`→`AirsideServer`).
- The **domain** is untouched: the `Comment` type/schema, `comments[]` fields, the `comments` Mongo database name, and "Add comment"/"Reply" UI copy stay.
- GitHub repo renamed in place `Airnauts/commenting-tool` → `Airnauts/airside` (auto-redirects).
- The old `@airnauts/comments-*` packages are **deprecated with a pointer** to their airside replacements — no shim, no republish.
- Versioning **continues the line**: the final `comments` release was `0.7.0`; airside debuts at `0.8.0` (one breaking minor under the pre-1.0 policy), same code and maturity.

**Consequences.** Existing embeds lose persisted state on upgrade (identity re-prompt, launcher position reset, one extra activation) — an accepted one-time cost vs. a permanent dual-read shim. Every internal `workspace:^` dep flips in one change, so the Changesets-only publish path is load-bearing (a hand `npm publish` would leak `workspace:^`). The one published consumer (lear-frontend) breaks and needs a cross-repo follow-up (new package names, `airsideKey`, `?airside-key`). ADR-0020's package-naming decision is superseded; its npm-scope and MIT/Changesets decisions stand.
```
